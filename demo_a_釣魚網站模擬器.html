🛡️ 你「本來應該」察覺的破綻（防禦重點）

破綻 01
網域名稱不對
真正的網域是「account-verify.xyz」，CloudVault 只是被塞在子網域裡騙你。永遠從右往左讀網域。
破綻 02
沒有 HTTPS 鎖頭
網址是 http:// 而非 https://，正規服務的登入頁幾乎不可能沒有加密。
破綻 03
「重新驗證身分」的急迫話術
製造急迫與恐懼，是社交工程最常見的觸發點，目的是讓你來不及思考。
破綻 04
原始碼可被任何人複製
攻擊者只要「另存網頁」或檢視原始碼，就能在幾分鐘內複製出視覺上一模一樣的假頁。

檢視原始碼 — 釣魚頁的核心其實只有幾行

下面是「擷取輸入」的關鍵程式。重點不是教你寫，而是讓你理解：視覺上像不像，跟安不安全完全是兩回事。任何頁面都能被複製，唯一可信的是網址與憑證機制。

// 攻擊者只需要監聽輸入框，就能拿到你打的每個字
// （本示範刻意「不」加任何網路傳送的程式碼）
emailInput.addEventListener('input', e => {
    const stolen = e.target.value;          // ← 你的帳號
    showOnAttackerPanel(stolen);            // 本示範：只顯示在本機
    // sendToAttacker(stolen);              // ← 真實攻擊會做這件事（本檔不實作）
});

pwdInput.addEventListener('input', e => {
    const stolenPwd = e.target.value;       // ← 你的密碼（明文）
    showOnAttackerPanel(stolenPwd);
});

防禦啟示：因為頁面本身永遠可被複製，判斷真偽唯一可靠的依據是「網址 + 來源驗證」，而不是「看起來像不像」。請改用密碼管理員（它會因網址不符而拒填）、硬體金鑰（FIDO2）等「綁定網域」的機制。
